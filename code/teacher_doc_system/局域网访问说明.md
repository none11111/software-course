-- 快速启动

--- 方法一：使用批处理文件（推荐）
1. 双击运行 `start_lan.bat`
2. 系统会自动启动服务器并显示访问地址

--- 方法二：使用Python脚本
```
python start_lan.py
```

--- 方法三：手动启动
```
- 设置环境变量
set ALLOWED_HOSTS=*
- 启动服务器
python manage.py runserver 0.0.0.0:8000
```

-- 访问地址

--- 本机访问
- http://localhost:8000
- http://127.0.0.1:8000

--- 局域网访问
- http://[您的IP地址]:8000
- 例如：http://192.168.1.100:8000

-- 获取本机IP地址

--- Windows系统
```
ipconfig
```
查找 "以太网适配器" 或 "无线局域网适配器" 下的 IPv4 地址

--- Linux/Mac系统
```
ifconfig
- 或
ip addr show
```

-- 防火墙配置

--- Windows防火墙
1. 打开"Windows Defender 防火墙"
2. 点击"允许应用或功能通过Windows Defender防火墙"
3. 点击"更改设置" -> "允许其他应用"
4. 添加Python.exe，勾选"专用"和"公用"

--- 路由器配置
如果无法访问，可能需要：
1. 检查路由器是否开启了AP隔离
2. 确保设备在同一网段
3. 检查防火墙设置

-- 安全注意事项

⚠️ 重要提醒：
- 当前配置仅适用于开发环境
- 生产环境请修改 `ALLOWED_HOSTS` 为具体的IP地址
- 建议在生产环境中使用HTTPS
- 定期更新密码和访问权限

-- 常见问题

--- 1. 无法访问
- 检查防火墙设置
- 确认IP地址是否正确
- 检查网络连接

--- 2. 端口被占用
```bash
- 查看端口占用
netstat -ano | findstr :8000
- 杀死进程
taskkill /PID [进程ID] /F
```

--- 3. 权限问题
- 确保以管理员身份运行
- 检查文件权限设置

-- 管理员账号
- 用户名：admin
- 密码：Admin123456

-- 技术支持
如遇到问题，请检查：
1. Django服务器是否正常启动
2. 网络连接是否正常
3. 防火墙设置是否正确
4. IP地址是否正确
